Class BasicEntityPersister

A BasicEntityPersiter maps an entity to a single table in a relational database.

A persister is always responsible for a single entity type.

EntityPersisters are used during a UnitOfWork to apply any changes to the persistent state of entities onto a relational database when the UnitOfWork is committed, as well as for basic querying of entities and their associations (not DQL).

The persisting operations that are invoked during a commit of a UnitOfWork to persist the persistent entity state are:
- Doctrine\ORM\Persisters\BasicEntityPersister::addInsert() : To schedule an entity for insertion. - Doctrine\ORM\Persisters\BasicEntityPersister::executeInserts() : To execute all scheduled insertions. - Doctrine\ORM\Persisters\BasicEntityPersister::update() : To update the persistent state of an entity. - Doctrine\ORM\Persisters\BasicEntityPersister::delete() : To delete the persistent state of an entity.

As can be seen from the above list, insertions are batched and executed all at once for increased efficiency.

The querying operations invoked during a UnitOfWork, either through direct find requests or lazy-loading, are the following:
- Doctrine\ORM\Persisters\BasicEntityPersister::load() : Loads (the state of) a single, managed entity. - Doctrine\ORM\Persisters\BasicEntityPersister::loadAll() : Loads multiple, managed entities. - Doctrine\ORM\Persisters\BasicEntityPersister::loadOneToOneEntity() : Loads a one/many-to-one entity association (lazy-loading). - Doctrine\ORM\Persisters\BasicEntityPersister::loadOneToManyCollection() : Loads a one-to-many entity association (lazy-loading). - Doctrine\ORM\Persisters\BasicEntityPersister::loadManyToManyCollection() : Loads a many-to-many entity association (lazy-loading).

The BasicEntityPersister implementation provides the default behavior for persisting and querying entities that are mapped to a single database table.

Subclasses can be created to provide custom persisting and querying strategies, i.e. spanning multiple tables.

Direct known subclasses

Doctrine\ORM\Persisters\AbstractEntityInheritancePersister, Doctrine\ORM\Persisters\UnionSubclassPersister, Doctrine\Tests\Mocks\EntityPersisterMock

Indirect known subclasses

Doctrine\ORM\Persisters\JoinedSubclassPersister, Doctrine\ORM\Persisters\SingleTablePersister

# update( object $entity )

Updates a managed entity. The entity is updated according to its current changeset in the running UnitOfWork. If there is no changeset, nothing is updated.

Updates a managed entity. The entity is updated according to its current changeset in the running UnitOfWork. If there is no changeset, nothing is updated.

The data to update is retrieved through Doctrine\ORM\Persisters\BasicEntityPersister::_prepareUpdateData(). Subclasses that override this method are supposed to obtain the update data in the same way, through Doctrine\ORM\Persisters\BasicEntityPersister::_prepareUpdateData().

Subclasses are also supposed to take care of versioning when overriding this method, if necessary. The Doctrine\ORM\Persisters\BasicEntityPersister::_updateTable() method can be used to apply the data retrieved from {@_prepareUpdateData} on the target tables, thereby optionally applying versioning.

# _prepareUpdateData( object $entity )

Prepares the changeset of an entity for database insertion (UPDATE).

Prepares the changeset of an entity for database insertion (UPDATE).

The changeset is obtained from the currently running UnitOfWork.

During this preparation the array that is passed as the second parameter is filled with <columnName> => <value> pairs, grouped by table name.

Example:

array(
   'foo_table' => array('column1' => 'value1', 'column2' => 'value2', ...),
   'bar_table' => array('columnX' => 'valueX', 'columnY' => 'valueY', ...),
   ...
)
